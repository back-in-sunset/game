## 评论功能集成测试描述

### 一、测试范围
- **核心功能**：发表评论、回复评论、点赞评论、查看评论列表（含分页/加载更多）
- **依赖模块**：用户认证、内容审核（可选）、消息通知、数据持久化
- **数据流**：前端请求 → API 网关 → 业务逻辑 → 数据库/缓存 → 返回结果

---

### 二、测试环境
- 后端服务（评论服务、用户服务）正常运行
- 关系型数据库（如 MySQL）存储评论正文、用户 ID、时间戳、点赞数
- 缓存（如 Redis）存储热评列表
- 前端页面（Web/移动端）集成真实 API

---

### 三、集成测试用例

#### 1. 发表评论（正向场景）
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 用户登录后，在输入框输入合法内容（如“杰克确实伟大”） | 提交按钮可点击 |
| 2 | 点击“发布” | 请求成功，返回 HTTP 201 |
| 3 | 刷新评论列表 | 新评论出现在列表顶部（按时间倒序） |
| 4 | 检查数据库 | `comment` 表中新增对应记录，`user_id`、`content`、`create_time` 正确 |
| 5 | 检查用户消息中心 | 未涉及他人时不触发通知；若 @ 了他人，应收到通知 |

#### 2. 回复评论（嵌套评论）
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 点击某条评论下的“回复”按钮 | 弹窗或内联输入框出现，自动填充 `@用户名 ` |
| 2 | 输入回复内容并提交 | 回复记录 `parent_id` 指向原评论 ID |
| 3 | 查询该原评论下的回复列表 | 回复按时间正序展示，缩进或特殊样式区分 |
| 4 | 删除原评论 | 所有子回复应级联删除或显示“评论已删除” |

#### 3. 点赞评论
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 未登录用户点击点赞 | 跳转登录页或提示登录 |
| 2 | 登录用户对评论 A 点赞 | 点赞按钮状态变为已点赞（高亮），点赞数 +1 |
| 3 | 再次点击同一评论 | 取消点赞，点赞数 -1 |
| 4 | 刷新页面 | 点赞状态和计数保持一致（持久化） |
| 5 | 并发点赞（同一用户两设备同时点赞） | 最终只计一次，数据库无重复记录 |

#### 4. 评论列表加载（分页/无限滚动）
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 请求第一页（`page=1&size=10`） | 返回前 10 条评论，含 total 总数 |
| 2 | 滚动到底部触发加载下一页 | 请求 `page=2`，新数据追加到列表末尾 |
| 3 | 当页面为最后一页时再次加载 | 不发送请求，或返回空数组，提示“没有更多” |
| 4 | 排序方式切换（按热度/时间） | 重新请求列表，数据顺序改变 |

#### 5. 删除评论（权限）
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 普通用户删除自己的评论 | 成功删除，数据库记录标记 `deleted=1` 或物理删除 |
| 2 | 普通用户删除他人评论 | 返回 403 Forbidden |
| 3 | 管理员删除任意评论 | 成功删除，前端列表移除该评论 |
| 4 | 删除后查看评论详情 | 显示“该评论已删除”占位符 |

#### 6. 内容审核（集成敏感词/反垃圾）
| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 发表包含敏感词的内容（如政治、色情） | 发布失败，提示“内容包含违规信息” |
| 2 | 发表含链接的评论（若策略禁止） | 被拦截或进入人工审核队列 |
| 3 | 短时间内频繁发表相同内容 | 触发频率限制，返回 429 Too Many Requests |

#### 7. 异常与边界
| 场景 | 输入 | 预期 |
|------|------|------|
| 内容为空 | 空白或仅空格 | 提示“评论不能为空”，不发送请求 |
| 超长内容 | 超过 1000 字符 | 后端截断或返回 400，前端应有字数计数器 |
| 特殊字符/表情 | 包含 `<>`、`<script>` | 经过 XSS 过滤，存储为纯文本或转义后展示 |
| 并发下的点赞数 | 100 人同时点赞 | 最终计数正确（使用乐观锁或原子操作） |

### 四、集成点验证矩阵

| 模块A | 模块B | 验证点 |
|-------|-------|--------|
| 前端 | 评论 API | 请求/响应格式、错误码处理、loading 状态 |
| 评论 API | 用户服务 | token 解析、用户是否存在、是否被禁言 |
| 评论 API | 数据库 | 事务回滚（如同时写评论和通知，一方失败则全失败） |
| 评论 API | 消息队列 | 异步发送通知（@提醒、点赞通知）最终一致性 |
| 评论 API | 缓存 | 发布新评论后，热评缓存失效并重建 |

---

### 五、测试数据准备
- 多个测试用户（普通用户、管理员、被禁言用户）
- 预置 30 条评论（含三级回复、不同点赞数）
- 敏感词词库（测试用）

---

### 六、通过标准
- 所有核心用例满足预期
- 并发场景下数据一致（无重复点赞、无丢失计数）
- 权限控制有效（无越权删除/修改）
- 异常输入不导致服务崩溃或数据损坏
